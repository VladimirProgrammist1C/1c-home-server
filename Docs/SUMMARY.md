## 🎯 Ключевые решения

### Почему Docker Desktop + WSL2?
- ✅ Проще для Windows-хоста
- ✅ Не требует отдельной ВМ
- ✅ Portainer работает из коробки
- ❌ Сложности с Docker socket на Windows

### Почему не Hyper-V для СУБД?
- ❌ Сложнее с сетью (NAT/bridge)
- ❌ Больше накладных расходов
-  Меньше интеграции с Windows

### Почему не Proxmox?
- ❌ Требует отдельного сервера
- ❌ Избыточно для домашнего использования
- ❌ Нужна дополнительная машина

### Почему `0.0.0.0` в docker-compose?
- ✅ Tailscale шифрует трафик → безопасно
- ✅ Не нужно настраивать проброс портов на роутере
- ✅ Доступ с любого устройства в сети Tailscale
- ⚠️ Не использовать без VPN!

### Почему именованные volumes?
- ✅ Переносимость между системами
- ✅ Проще бэкапить (`docker volume inspect`)
- ✅ Не зависят от путей на хосте

### Почему Tailscale?
- ✅ Бесплатно для личного использования
- ✅ WireGuard шифрование
- ✅ Не требует проброса портов
- ✅ Работает из любых сетей (WiFi + мобильные данные)

### Почему внедрён и протестирован Агент сервера 1С?
- ✅ Многопользовательский режим: Проверено, что несколько сеансов 1С работают одновременно без конфликтов.
- ✅ Параллельная работа: Убедились, что Конфигуратор и режим Предприятия функционируют параллельно.
- ✅ Лицензирование: Лицензия developer.1c.ru корректно захватывается в клиент-серверном режиме.
- ✅ Управление: Консоль администрирования работает стабильно (после переустановки компонента).
- ✅ Готовность к росту: Инфраструктура теперь готова к подключению дополнительных разработчиков или запуску фоновых заданий.
> 💡Тестирование подтвердило, что домашний сервер полностью готов к работе в режиме полноценного сервера приложений 1С, а не просто файловой базы.

### Почему Windows 11 Pro ИЛИ Windows Server для терминала?
| Критерий | 11 Pro | Server |
|----------|--------|--------|
| Лицензия | ✅ Уже на Geekom | ✅ Есть дистрибутив |
| ОЗУ | ~2-3 ГБ | ~4-6 ГБ |
| RDP-сессии | 1 одновременная | Множество (с настройкой) |
| Водяной знак | Нет | Есть (косметика, не влияет) |
| Для 1 разработчика | ✅ Достаточно | ✅ Тоже подойдёт |

## ⚠️ Проблемы и решения

| Проблема | Решение | Время потери |
|----------|---------|--------------|
| Portainer не подключается к Docker | Использовать `//./pipe/docker_engine` + API на порту 2375 | 90 минут |
| Доступ к веб-панелям извне | Порты `0.0.0.0` + Tailscale (безопасно) | 15 минут |
| Таблицы в Markdown не работают | Одна строка таблицы = одна строка кода | 30 минут |
| Git rebase + Vim | Настроить VS Code как редактор (`git config --global core.editor "code --wait"`) | 15 минут |
| Ошибка локали PostgreSQL | Создавать базу с явной локалью через `TEMPLATE=template0` | 45 минут |
| Пароль PostgreSQL не меняется | `ALTER USER` вместо перезапуска контейнера | 10 минут |
| Символические ссылки для 1С | Скрипт с бэкапом перед созданием ссылки | 20 минут |
| Ollama на Windows без GPU | Запуск через WSL2 или только CPU | 45 минут |
| Консоль администрирования не запускается | Переустановить компонент "Администрирование сервера" | 15 минут |

> 💡 **Почему GitHub?** Аккаунт GitHub используется как провайдер авторизации для входа в **Tailscale**. Включение двухфакторной аутентификации (2FA) на GitHub критически важно: если злоумышленник получит доступ к аккаунту GitHub, он сможет войти в вашу приватную сеть Tailscale и получить доступ ко всем сервисам (Portainer, pgAdmin, БД). Поэтому защита GitHub = защита всей инфраструктуры.
