## 🎯 Ключевые решения

### Почему Docker Desktop + WSL2?
- Проще для Windows-хоста
- Не требует отдельной ВМ
- Portainer работает из коробки

### Почему не Hyper-V?
- Сложнее с сетью
- Меньше интеграции с Windows

### Почему не Proxmox?
- Требует отдельного сервера
- Избыточно для домашнего использования

## 🖥️ Обсуждение ресурсов и характеристик Geekom A9 MAX

### 🔹 Технические характеристики (обсуждались):

| Компонент | Значение | Комментарий |
|-----------|----------|-------------|
| **Процессор** | AMD Ryzen AI 9 HX 370 | 12 ядер / 24 потока, до 5.1 ГГц |
| **Встроенная графика** | AMD Radeon 890M | iGPU, использует системную ОЗУ |
| **ОЗУ (штатно)** | 32 ГБ DDR5 | Планируется апгрейд до 64–128 ГБ |
| **Накопитель** | 1 ТБ NVMe SSD | Инфраструктура на диске E: (~1 ТБ свободно) |
| **NPU** | AMD XDNA (до 50 TOPS) | Для локальных AI-задач (Ollama) |
| **ОС** | Windows 11 Pro | Поддержка Hyper-V, WSL2, Docker Desktop |

### 🔹 Ключевые выводы по ресурсам:

✅ **Для 1С-разработки (1 пользователь, несколько контуров):**
- 32 ГБ ОЗУ **достаточно** для старта
- Рекомендуется апгрейд до **64 ГБ** для комфортной работы с несколькими ИБ + Docker + LLM

✅ **Для Docker-инфраструктуры:**
- PostgreSQL + Portainer + pgAdmin занимают ~2–4 ГБ ОЗУ
- Образы контейнеров: ~1–2 ГБ на диске
- WSL2 требует выделения 8–16 ГБ ОЗУ (настраивается в `.wslconfig`)

✅ **Для локальных LLM (Ollama):**
- Модель `qwen3:8b` (5.9 ГБ) работает, но **только на CPU** (Ollama на Windows не поддерживает GPU AMD без WSL2)
- Инференс: ~10–30 сек на ответ (против 1–3 сек в облаке)
- Для ускорения: запуск Ollama в WSL2 + настройка ROCm (сложно, нестабильно на Windows)

✅ **Температурный режим:**
- Мини-ПК компактный, но система охлаждения справляется
- При длительной нагрузке (сборка Docker-образов, инференс LLM) — мониторить температуры

## 🗂️ Основные темы обсуждения (хронологически)

### 🔹 22 марта: Старт проекта
- **Выбор стратегии развёртывания:** Hyper-V vs WSL2 vs Proxmox → остановились на **Docker Desktop + WSL2** (проще для Windows)
- **Целевая архитектура:**
  ```
  ┌─────────────────────────┐
  │ 1С:Предприятие (хост)   │
  ├─────────────────────────┤
  │ Docker (WSL2 backend)   │
  │ ├─ PostgreSQL 1C        │
  │ ├─ Portainer            │
  │ └─ pgAdmin              │
  └─────────────────────────┘
  ```
- **Первые проблемы:** MagicDNS не работал для доступа к расшаренным папкам (`a9-max/Repo1C` → 404)

### 🔹 23 марта: Локальные LLM и AI-интеграции
- **Ollama + Mini AI 1C:** установка, выбор модели (`qwen3:8b`), проблемы с производительностью
- **GPU-ускорение:** выяснили, что Ollama на Windows **не поддерживает AMD GPU** нативно → только CPU или WSL2+ROCm (сложно)
- **MCP-интеграции:** тестирование `mini-ai-1c`, `1c-mcp-toolkit`, `1c-ai-feature-dev-workflow` → остановились на проверенном `mini-ai-1c`
- **Размещение данных:** все AI-модели → `E:\1C_Infrastructure\AI` (экономия места на C:)

### 🔹 26 марта: Мессенджер и Docker
- **VoceChat:** развёртывание локального мессенджера (аналог Telegram) для семейного чата (до 25 устройств) + ZeroTier для доступа извне
- **Docker-каталоги:** настройка томов (`volumes`) для изоляции данных 1С и мессенджера
- **Хранение:** инфраструктура на диске E: (1 ТБ свободно), C: — только система и программы

### 🔹 27 марта: Финализация инфраструктуры СУБД
- **PostgreSQL 1C:** сборка образа из официального дистрибутива 1С (`postgresql_18.1_2_ubuntu_22.04_x86_64_package.tar.bz2`)
- **docker-compose.yml:** оркестрация сервисов (PostgreSQL + Portainer + pgAdmin)
- **Portainer:** 90 минут отладки подключения к Docker API на Windows → решение: `host.docker.internal:2375`
- **pgAdmin:** настройка подключения к контейнеру PostgreSQL (Host: `postgres` — имя сервиса!)

### 🔹 29 марта: Документирование и организация
- **README.md:** создание полной документации с таблицами, инструкциями, page-break для PDF
- **TIMING.md:** детальный учёт времени по этапам (~4.5 часа чистой работы)
- **Git:** настройка репозитория, Conventional Commits, интерактивный rebase для приведения истории в порядок
- **Финальный статус:** инфраструктура СУБД готова на 100%, документы закоммичены, репозиторий на GitHub

3. **Git rebase** — интерактивное редактирование истории (Vim vs VS Code)
