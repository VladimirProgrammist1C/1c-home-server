## 🎯 Ключевые решения

### Почему Docker Desktop + WSL2?
- ✅ Проще для Windows-хоста
- ✅ Не требует отдельной ВМ
- ✅ Portainer работает из коробки
- ❌ Сложности с Docker socket на Windows (решено через TCP API)

### Почему не Hyper-V для СУБД?
- ❌ Сложнее с сетью (NAT/bridge)
- ❌ Больше накладных расходов
- ❌ Меньше интеграции с Windows

### Почему не Proxmox?
- ❌ Требует отдельного сервера
- ❌ Избыточно для домашнего использования
- ❌ Нужна дополнительная машина

### Почему `0.0.0.0` в docker-compose?
- ✅ Tailscale шифрует трафик → безопасно
- ✅ Не нужно настраивать проброс портов на роутере
- ✅ Доступ с любого устройства в сети Tailscale
- ⚠️ **Не использовать без VPN!**

### Почему именованные volumes?
- ✅ Переносимость между системами
- ✅ Проще бэкапить (`docker volume inspect`)
- ✅ Не зависят от путей на хосте

### Почему Tailscale?
- ✅ Бесплатно для личного использования
- ✅ WireGuard шифрование
- ✅ Не требует проброса портов
- ✅ Работает из любых сетей (WiFi + мобильные данные)

### Почему СУБД в Docker, а 1С на хосте?
- ✅ Лицензия developer.1c.ru привязана к железу (хост)
- ✅ СУБД в Docker — изоляция, чистота, легко пересоздать
- ✅ Для лицензии не важно, где СУБД (хост или Docker) — платформа на хосте
- ✅ Проще с отладкой и мониторингом на Windows
- ⚠️ Меньше изоляции для 1С (но для дома — не критично)

### Почему русская локаль в кластере PostgreSQL — критична?
- ✅ 1С:Предприятие требует `ru_RU.UTF-8` для корректной работы
- ✅ Без правильной локали базы не создаются ни через консоль, ни через Обновлятор
- ✅ Решение: настроить `entrypoint.sh` с `LC_COLLATE` и `LC_CTYPE` при сборке образа

### Почему Агент сервера 1С протестирован?
- ✅ Многопользовательский режим работает (несколько сеансов одновременно)
- ✅ Конфигуратор + Предприятие работают параллельно
- ✅ Лицензия developer.1c.ru захватывается корректно
- ✅ Консоль администрирования работает (после переустановки компонента)

### Почему Обновлятор 1С от Владимира Милькина?
- ✅ Готовое GUI-решение (не нужно писать скрипты)
- ✅ Встроенный планировщик заданий (интеграция с Windows Task Scheduler)
- ✅ Поддержка всех операций: архивация, обновление, проверка, восстановление
- ✅ Работает с клиентскими утилитами из pgAdmin runtime

### Почему Prometheus + Grafana + Blackbox?
- ✅ Современный стек мониторинга
- ✅ Автоматические алерты и уведомления
- ✅ Blackbox для HTTP-проверок (не требуют метрик от сервисов)
- ✅ Интеграция с VoceChat для локальных уведомлений
- ❌ cAdvisor на Docker Desktop работает только для метрик хоста

### Почему VoceChat для уведомлений?
- ✅ Локальный мессенджер (не требует интернета)
- ✅ Простая интеграция через webhook
- ✅ Не нужно регистрировать Telegram-бота
- ✅ Полный контроль над данными

## ⚠️ Проблемы и решения

| Проблема | Решение | Время потери |
|----------|---------|--------------|
| Portainer не подключается к Docker API | Использовать `//./pipe/docker_engine` + API на порту 2375 | 90 минут |
| Доступ к веб-панелям извне | Порты `0.0.0.0` + Tailscale (безопасно) | 15 минут |
| Ошибка локали при создании базы 1С | Пересобрать контейнер с `ru_RU.UTF-8` в `entrypoint.sh` | 45 минут |
| Ссылки в Markdown не работали (регистр) | Исправить `docs` → `Docs` в путях | 30 минут |
| Синтаксис Markdown-ссылок | Убрать лишние скобки `[[...](...)](...)` | 20 минут |
| Обновлятор не находил утилиты PostgreSQL | Указать путь к `pgAdmin 4\runtime` | 40 минут |
| Консоль администрирования не запускалась | Переустановить компонент "Администрирование сервера" | 15 минут |
| cAdvisor не видит контейнеры | Docker Desktop limitation, использовать Blackbox | 45 минут |
| pgAdmin/Grafana 401/302 ошибки | Health endpoints + custom blackbox.yml | 60 минут |
| Уведомления JSON вместо текста | text/plain + шаблоны Grafana | 30 минут |
| Prometheus relabel_configs синтаксис | Использовать `target_label` вместо `target_param` | 90 минут |
| Сети Docker (monitoring vs 1c-infrastructure) | Добавить все сервисы в обе сети | 45 минут |
| Конфликт портов PostgreSQL | Остановить локальный PG, отключить службу | 30 минут |
| Алерты не сбрасываются | Перезапуск Prometheus, проверка queries | 30 минут |
| **ИТОГО** | | **~8 часов 30 мин (~27%)** |
