### 🔹 Этап 7: Настройка Portainer (90 минут) ⚠️ САМЫЙ СЛОЖНЫЙ

⏱️ 00:00-00:20 — Первый вход, создание пользователя (пароль: admin1234567)  
⏱️ 00:20-00:40 — Попытки подключиться к Docker (Socket vs API)  
⏱️ 00:40-01:00 — Включение TCP API в Docker Desktop (порт 2375)  
⏱️ 01:00-01:20 — Проблема: localhost внутри контейнера ≠ localhost на хосте  
⏱️ 01:20-01:30 — Решение: `host.docker.internal:2375`

**Время:** 90 минут  
**Сложность:** ⭐⭐⭐⭐⭐  
**Итераций:** 20+

**Ошибки:**
- Неправильный путь к socket (Linux vs Windows)
- Portainer ищет `/var/run/docker.sock`, а не `//./pipe/...`
- Двойной протокол: `http://tcp://...`
- Вкладки Linux/Windows в Portainer

**Решение:**
- Подключение по: `host.docker.internal:2375`
- Portainer видит все контейнеры, volumes, сети

## 🎯 Ключевые инсайты для будущего

### ⚡ Что ускорило бы процесс:

**Portainer: сразу использовать API, а не Socket**
```yaml
# Docker API URL: host.docker.internal:2375
# TLS: OFF
```
**Экономия:** ~60 минут!

**Использовать готовый шаблон docker-compose для Windows**
```yaml
volumes:
  - //./pipe/docker_engine:/var/run/docker.sock  # Windows → Linux
```
**Экономия:** ~30 минут

**Проверять синтаксис перед сборкой**
```powershell
docker-compose config  # Проверка docker-compose.yml
```
**Экономия:** ~15 минут

**Сразу использовать Conventional Commits**
```
Add: новый файл
Fix: исправление
Docs: документация
Chore: конфигурации
```
**Экономия:** ~15 минут (не нужно переделывать rebase)

**Настроить VS Code как редактор для Git**
```powershell
git config --global core.editor "code --wait"
```
**Экономия:** ~10 минут (не мучиться с Vim)

**Таблицы Markdown: сразу форматировать правильно**
```markdown
| Заголовок 1 | Заголовок 2 |
|-------------|-------------|
| Ячейка 1    | Ячейка 2    |
```
**Экономия:** ~30 минут

**Создавать базу с локалью сразу через SQL**
```sql
CREATE DATABASE "MyBase" WITH 
  LC_COLLATE='ru_RU.UTF-8' 
  LC_CTYPE='ru_RU.UTF-8' 
  TEMPLATE=template0 
  ENCODING='UTF8';
```
**Экономия:** ~45 минут
