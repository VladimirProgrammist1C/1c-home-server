## 2. Установка <a id="2-установка"></a>

1. Скопируйте файл расширения `build/ВЧ_ИнтеграцияСVoceChat_v1.0.cfe` на сервер 1С.
2. Откройте **Конфигуратор** основной базы.
3. Перейдите: `Конфигурация` → `Расширения конфигурации` → `Добавить`.
4. Выберите файл `.cfe` и нажмите **Открыть**.
5. Нажмите **F7** для обновления конфигурации базы данных.
6. Закройте Конфигуратор и запустите базу в режиме **1С:Предприятие**.

> 💡 **Важно:** Расширение не изменяет типовые объекты ЗУП. При обновлении конфигурации от вендора конфликтов не возникнет.

## 3. Настройка интеграции <a id="3-настройка-интеграции"></a>

### 3.1. Назначение прав
1. Откройте `Администрирование` → `Настройки пользователей и прав` → `Группы доступа`.
2. Создайте новую группу папку с именем `Интеграция с VoceChat`.
3. Внутри группы создайте профили доступа:
   - **`Администратор интеграции с VoceChat`** — добавьте роль `ВЧ_АдминистраторИнтеграции` и `ВЧ_ЧтениеИнтеграции`
   - **`Чтение интеграции с VoceChat`** — добавьте роль `ВЧ_ЧтениеИнтеграции`
4. Включите пользователей в соответствующие профили.

### 3.2. Заполнение параметров
1. Откройте общую форму: `ВЧ_НастройкиИнтеграции`.
2. **⚠️ Ключевой шаг:** Сначала включите переключатель `ВЧ_ИспользоватьИнтеграцию`.  
   *Только после этого поля ввода станут доступными.*
3. Заполните параметры:

| Параметр | Пример значения | Описание |
|----------|----------------|----------|
| `ВЧ_URLСервера` | `localhost` | IP или домен сервера VoceChat (без `http://`) |
| `ВЧ_ПортСервера` | `3001` | Порт, на котором слушает VoceChat |
| `ВЧ_BotAPIKey` | `x-api-key-xxxx...` | Ключ бота из админки VoceChat (маскируется) |
| `ВЧ_IDКанала` | `1` | Числовой ID целевого канала в VoceChat |
| `ВЧ_ТаймаутHTTP` | `30` | Таймаут соединения в секундах |
| `ВЧ_МаксимальноеКоличествоПопыток` | `3` | Сколько раз повторять отправку при сбое |
| `ВЧ_ОтветственныйКадровик` | `Иванов И.И.` | Пользователь 1С по умолчанию |

4. Нажмите **Записать и закрыть**.

## 4. Проверка работоспособности <a id="4-проверка-работоспособности"></a>

### 4.1. Тестовая отправка
1. Выполните команду `ВЧ_ТестированиеОтправки` (из панели навигации или через `Файл` → `Открыть`).
2. В открывшейся обработке заполните параметры подключения (адрес, порт, Bot API Key, ID канала).
3. В поле `Текст сообщения` введите тестовое сообщение (по умолчанию: `Тест из 1С:Предприятия`).
4. Нажмите кнопку «Выполнить тест».
5. Проверьте канал VoceChat — должно прийти сообщение с указанным текстом.

### 4.2. Проверка на реальном документе
1. Создайте новый документ `Прием на работу`.
2. Заполните обязательные поля: Сотрудник, Подразделение, Должность.
3. Проведите документ.
4. Подождите до 30 секунд (интервал регламентного задания).
5. Откройте регистр `ВЧ_ОчередьУведомлений` (через `Все функции` → `Регистры сведений`).  
   *Статус записи должен измениться на `Отправлено`.*
6. Убедитесь, что уведомление пришло в VoceChat.

### 4.3. Журнал регистрации
Откройте `Администрирование` → `Журнал регистрации`.  
Отфильтруйте по событию `ВЧ_Интеграция`. Должны присутствовать записи:
- `Задача на отправку уведомления создана`
- `Уведомление успешно отправлено`

## 5. Устранение неполадок <a id="5-устранение-неполадок"></a>

| Симптом | Возможная причина | Решение |
|---------|------------------|---------|
| Поля настроек заблокированы | Интеграция не включена | Включите `ВЧ_ИспользоватьИнтеграцию` |
| Ошибка `401 Unauthorized` | Неверный `BotAPIKey` | Проверьте ключ в админке VoceChat |
| Ошибка `404 Not Found` | Неверный `IDКанала` | Уточните ID канала в настройках VoceChat |
| Ошибка `Таймаут / 503` | Сервер VoceChat недоступен или блокируется фаерволом | Проверьте сетевой доступ с сервера 1С на порт VoceChat |
| Уведомления не уходят, статус `Ожидает` | Регламентное задание отключено или зависло | `Администрирование` → `Регламентные задания` → проверьте `ВЧ_ОбработчикОчередиУведомлений` |
| Дублируются сообщения | Документ перепроводится многократно | Интеграция реагирует только на **первое проведение**. Повторные отправки игнорируются |

## 6. Обслуживание и обновление <a id="6-обслуживание-и-обновление"></a>

### 6.1. Очистка очереди
Записи со статусом `Ошибка` не удаляются автоматически. Для очистки:
1. Откройте регистр `ВЧ_ОчередьУведомлений`.
2. Отберите записи по статусу `Ошибка` или старые по дате.
3. Удалите выбранные записи или измените статус вручную.

### 6.2. Обновление расширения
1. Скачайте новую версию `.cfe`.
2. В Конфигураторе: `Расширения конфигурации` → выберите текущее → `Заменить`.
3. Обновите БД (F7).
4. Параметры настроек сохраняются между версиями.

### 6.3. Резервное копирование
При бэкапе базы 1С данные расширения (константы, регистр очереди) сохраняются автоматически. Дополнительных действий не требуется.
